## ðŸ› ï¸ Step-by-Step Deployment

### 1. Build Frontends Locally
Hostinger's RAM is often limited. Build everything on your computer first:

1. Open a terminal in the project root.
2. Run:
   ```bash
   cd backend
   npm install
   npm run build:full
   ```
   *This command now builds everything (Dashboard, Verifier, and Backend).*

### 2. Upload Files
Upload the entire project structure as shown above using FTP or File Manager.

### 3. Configure `.env`
Create a `.env` file at the **Project Root** (outside the `backend` folder). 
The backend is now programmed to search for it automatically!

```env
JWT_SECRET="your_secret"
PORT=5000
DATABASE_URL="postgresql://..."
```

### 4. Install & Run on Hostinger
1. In the Node.js Selector, click **Run NPM Install**.
   - *This will automatically trigger `npm run build` which now includes `prisma generate`.*
2. Click **Start App**.

## ðŸ” Troubleshooting
- **App won't start?** Make sure the **Application Startup File** is set to `backend/dist/index.js`.
- **Database errors?** Ensure your `DATABASE_URL` uses the correct external address if Supabase/Cloud DB is used.
- **Port issue?** Hostinger usually manages the port, but our app defaults to `process.env.PORT` which is fine.
